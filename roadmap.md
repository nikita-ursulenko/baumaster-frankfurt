# ROADMAP

## Текущая задача: Исправление отображения форматированного текста в блоге

**Цель**: Сделать скриншоты отформатированного текста в админке и фронтенде, выявить проблемы с отображением HTML-контента и исправить их

### Задачи:

- [ ] **Скриншот отформатированного текста в админке**
  - URL: `http://localhost/admin/blog.php`
  - Сделать скриншот списка статей с HTML-контентом
  - Сделать скриншот редактора при редактировании статьи
  - Зафиксировать как отображается отформатированный текст в админке

- [ ] **Скриншот отображения на фронтенде**
  - URL: `http://localhost/blog_post.php?slug=kak-vybrat-pravilnye-materialy-dlya-remonta-vannoy-komnaty`
  - Сделать скриншот страницы статьи
  - Зафиксировать проблему с отображением простого текста вместо HTML-разметки

- [ ] **Анализ проблемы**
  - Сравнить скриншоты админки и фронтенда
  - Выявить причину неправильного отображения HTML-контента
  - Определить где теряется форматирование

- [ ] **Исправление отображения на фронтенде**
  - Найти код рендеринга статей блога
  - Исправить вывод HTML-контента (убрать экранирование)
  - Добавить CSS стили для форматированного текста
  - Проверить работу WYSIWYG редактора

- [ ] **Проверка исправлений**
  - Сделать новые скриншоты после исправлений
  - Убедиться что HTML-контент отображается корректно
  - Проверить работу на разных статьях

## Ожидаемый результат

- HTML-контент статей блога должен отображаться с правильным форматированием на фронтенде
- Заголовки, списки, абзацы должны иметь соответствующие стили
- WYSIWYG редактор должен корректно сохранять и отображать форматирование

## Файлы для проверки

- `blog_post.php` - страница отдельной статьи
- `ux/layout.php` - базовый layout сайта
- `ux/components.php` - компоненты для отображения контента
- `admin/blog.php` - управление статьями в админке
- `assets/css/style.css` - стили для контента
